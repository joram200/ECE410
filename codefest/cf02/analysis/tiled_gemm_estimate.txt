--------------------------------------------------------------------
  FLOPs per update()         : 230
  AI ideal (tiled GEMM)      : 0.4637 FLOPs/byte
  Persistent data footprint  : 496 B  (fits in L1 — tile = full matrix)
  Peak DP compute            : 1177.6 GFLOP/s  (AVX-512, 8c, 4.6 GHz)

====================================================================
TILED GEMM ESTIMATE  (register-blocked, L1-resident)
--------------------------------------------------------------------
  For n=3, m=1: the 3×3 matrices are smaller than one AVX-512 register.
  A register-blocked tiled GEMM loads each persistent tensor once from L1,
  holds all intermediates in registers, and writes outputs back to L1.
  This realises the ideal AI and is bounded by L1 read bandwidth.

  Memory traffic (ideal)     : 496 B  (inputs) + 128 B  (outputs) = 496 B total
  L1 bandwidth (practical)   : 235 GB/s
  Memory-bound time          : 496 B / 235 GB/s = 2.11 ns
  Compute-bound time         : 230 FLOPs / 1177.6 GFLOP/s = 0.195 ns
  Bottleneck                 : MEMORY (L1)  — AI 0.4637 < ridge 5.01 FLOPs/B

  ┌─────────────────────────────────────────────────────────┐
  │  Attainable FLOP/s  ≈  109.0 GFLOP/s          │
  │  Time per update()  ≈  2.11 ns                    │
  └─────────────────────────────────────────────────────────┘

COMPARISON
--------------------------------------------------------------------
  Scenario                                      GFLOP/s    Time/update
  ──────────────────────────────────────── ──────────── ──────────────
  Python/NumPy baseline (measured)               0.0028        81.70 µs
  Tiled GEMM — DRAM bound (no reuse)               4.74        48.57 ns
  Tiled GEMM — L3 bound                           20.87        11.02 ns
  Tiled GEMM — L2 bound                           29.68         7.75 ns
  Tiled GEMM — L1 bound (target)                 108.97         2.11 ns
  Peak compute ceiling                           1177.6       0.195 ns

  Speedup vs Python baseline  : 38708×
  Efficiency vs peak compute  : 9.3%

KEY TAKEAWAYS
--------------------------------------------------------------------
  1. Even with perfect tiled GEMM (register-blocked, L1-resident), the
     KF update() remains MEMORY-BOUND (AI 0.4637 << ridge 5.01).
  2. The L1-bound ceiling is ~109 GFLOP/s — 9.3% of peak DP compute.
  3. To become compute-bound, AI must exceed the L1 ridge point
     (5.01 FLOPs/byte), requiring ~11× more FLOPs per
     byte loaded — e.g., batching many filter instances over shared A/C.
  4. Practical tiled GEMM in C achieves ~80% of the L1 ceiling =>
     ~87 GFLOP/s, still a 30966× improvement over baseline.
