====================================================================
Peak FLOP/s & Memory Bandwidth: RVfpgaEL2 on Nexys A7
====================================================================
PLATFORM
--------------------------------------------------------------------
  Core            : SweRV EL2 (VeeR EL2), in-order, 4-stage, single-issue
  FPGA            : Xilinx Artix-7 XC7A100T  (Nexys A7-100T)
  Clock frequency : 50 MHz  (VeeRwolf / RVfpga default)
  ISA             : RV32IMC  (NO hardware FPU in standard RVfpga build)
  DCCM            : 32–64 KB, 64-bit AXI4 bus
  DDR2            : MT47H64M16HR-25, 16-bit, 667 MT/s

====================================================================
PEAK INTEGER THROUGHPUT
--------------------------------------------------------------------
  1 core × 50 MHz × 1 issue/cycle
  = 50 MIPS

====================================================================
PEAK FLOP/s
--------------------------------------------------------------------
Case A — Standard RVfpga EL2  (RV32IMC, software FP emulation)
  Soft FMADD cost......................... ~70 integer instructions
  FMADDs per second....................... 714.3 k/s
  Peak SP FLOP/s (soft-float)............. 1.43 MFLOP/s

Case B — Hypothetical hardware FPU  (F extension, 1 FMADD/cycle)
  FLOPs per cycle per core................ 2
  Peak SP FLOP/s (hw FPU)................. 100.00 MFLOP/s

====================================================================
MEMORY BANDWIDTH
--------------------------------------------------------------------
1. DCCM (on-chip BRAM, 64-bit AXI4)
   Read  (1 bus transfer/cycle)............ 400.0 MB/s
   Write (1 bus transfer/cycle)............ 400.0 MB/s
   Combined (dual-port peak)............... 800.0 MB/s

2. DDR2  (MT47H64M16HR-25, 16-bit × 667 MT/s)
   Theoretical peak........................ 1.334 GB/s
   Practical (~50% MIG efficiency)          667.0 MB/s

====================================================================
ROOFLINE RIDGE POINTS  (AI = Peak FLOP/s / Peak BW)
--------------------------------------------------------------------
  Soft-FP  vs DCCM (read)   : 0.0036 FLOPs/byte
  HW-FPU   vs DCCM (read)   : 0.2500 FLOPs/byte
  HW-FPU   vs DDR2 (pract.) : 0.1499 FLOPs/byte

====================================================================
KALMAN FILTER CONTEXT
--------------------------------------------------------------------
  KF update() FLOPs         : 230
  KF arithmetic intensity   : 0.4637 FLOPs/byte (ideal)

  Estimated update() time on RVfpgaEL2 (soft-float):
    ~230/2 FMADDs × 70 cycles = 8050 cycles
    @ 50 MHz → 161.00 µs  (~0.16 ms)

  Compare: i7-11800H achieved ~38.89 µs/update (Python/numpy)
           RVfpgaEL2 estimate: ~161 µs/update (soft-float C)
           Slowdown factor   : ~4×

  Note: KF AI (0.46 FLOPs/byte) is ABOVE the ridge point for soft-FP
  on DCCM, meaning the filter is compute-bound on RVfpgaEL2 (not
  memory-bound), unlike on the i7-11800H.
